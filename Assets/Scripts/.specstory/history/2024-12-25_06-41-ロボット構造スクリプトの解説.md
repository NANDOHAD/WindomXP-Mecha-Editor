## ロボット構造スクリプトの解説 (2024/12/25 6:41:54)

_**Assistant**_

`RoboStructure.cs`は、Unityの`MonoBehaviour`を継承したクラスで、ロボットの構造を構築し、アニメーションやポーズの設定、モデルのインポート、パーツの追加・削除・名前変更などを行うスクリプトです。以下にその主要な機能を簡単に解説します。

### 主なフィールド
- `root`: ロボットのルートとなる`GameObject`。
- `parts`: ロボットの各パーツを格納するリスト。
- `hod`: ロボットの構造情報を保持する`hod2v0`オブジェクト。
- `ani`: アニメーション情報を保持する`ani2`オブジェクト。
- `Importer`: モデルをインポートするための`AssimpImporter`オブジェクト。
- `transcoder`: 暗号化されたデータを扱うための`CypherTranscoder`オブジェクト。

### 主なメソッド
- `Start()`: 初期化処理を行い、`transcoder`をインスタンス化します。
- `Update()`: 毎フレーム呼び出されるが、現在は何も処理していません。
- `buildStructure(hod2v0 Robo)`: 指定された`hod2v0`オブジェクトを基にロボットの構造を構築します。パーツを生成し、階層構造を設定します。
- `ImportModel(GameObject GO, string file)`: 指定されたファイルからモデルをインポートし、`GameObject`に適用します。
- `ImportModelEncrypted(GameObject GO, string file)`: 暗号化されたファイルからモデルをインポートし、`GameObject`に適用します。
- `setPose(hod2v0 pose)`: 指定されたポーズをロボットに適用します。
- `setPose(hod2v1 pose)`: `hod2v1`形式のポーズを適用します。
- `setPose(int AnimID, int HodID)`: アニメーションIDとフレームIDを指定してポーズを適用します。
- `updateHod(int AnimID, int HodID)`: 指定されたアニメーションとフレームに基づいてロボットの状態を更新します。
- `updatePart(...)`: 指定されたパーツの状態を更新します。オーバーロードされた複数のバージョンがあります。
- `addPart(string partName, int parent)`: 新しいパーツを追加し、構造を再構築します。
- `removePart(int index)`: 指定されたインデックスのパーツを削除します。
- `renamePart(int index, string name)`: 指定されたインデックスのパーツの名前を変更します。
- `createHod2v1()`: 現在のロボットの状態を`hod2v1`形式でコピーします。
- `saveAni()`: アニメーションデータを保存します。
- `saveHOD1()`: `hod1`形式でロボットの構造を保存します。

このスクリプトは、ロボットの構造を動的に構築し、アニメーションやポーズの管理を行うための機能を提供しています。
